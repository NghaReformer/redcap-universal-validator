# Tests — dual-runtime parity

Both engines are checked against one fixture generated by the Python source of
truth (`qrcode_generation/check_characters.py`). If either drifts, the test fails.

## Run

```bash
node tests/parity_js.cjs      # JS engine (js/engine.js) vs fixture
php  tests/parity_php.php      # PHP port (php/CheckCharacter.php) vs fixture
```

Each prints the algorithms covered and the mismatch count, and exits non-zero on
any mismatch. Both cover the same 420 `compute` rows across all 11 algorithms
(damm, verhoeff, luhn, none, and the seven ISO/IEC 7064 variants).

## `check_fixture.json`

Generated by `playground/gen_check_fixture.py` in the `qrcode_generation` repo.
It carries three sections; the parity tests use `compute` (`{name, payload,
check}` rows). To refresh it after an engine change, regenerate it there and copy
it here — see [`../js/README.md`](../js/README.md).

## CI

`.github/workflows/parity.yml` runs both tests on every push and pull request
(Node 20, PHP 8.1). Green CI means this module's client and server engines agree
with each other and with the ID generator.

## Note on the PHP port

`php/CheckCharacter.php` is a direct transliteration of the Python engine. Its
lookup tables (Damm, Verhoeff), alphabet constants, and ISO parameters have been
diffed against the authoritative Python values; the algorithm math is shared with
the JS engine, which passes 420/420. `parity_php.php` is the authoritative
end-to-end check and runs in CI.
